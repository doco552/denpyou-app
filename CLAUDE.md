# 仕入れ伝票 自動入力アプリ 開発指示書

## アプリの目的

リサイクルショップが仕入れ市場で受け取る「買伝票」を撮影し、
その画像から商品データを自動で読み取り、Excelファイルとして出力する
スタンドアロンの業務ツールを開発してください。

在庫管理システムとは完全に独立したアプリです。
事務処理として「伝票をExcelに変換する」ことだけを目的とします。

---

## 技術スタック

- 言語: Python 3.9+
- UI フレームワーク: Streamlit（ブラウザで動くシンプルなUI）
- 画像解析 AI: Google Gemini API（google-genai ライブラリ）
- 出力形式: Excel（.xlsx）
- 環境変数管理: python-dotenv（.env ファイルでAPIキー管理）

## 出力するExcelの列構成（最重要）

ユーザーがすでに使用している既存のExcel帳票に合わせること。
出力する列は以下の6列のみ。余計な列は追加しないこと。

| 列 | ヘッダー名 | 内容 |
|---|---|---|
| A | 年月日 | 仕入日（YYYY/MM/DD形式） |
| B | 場所 | 仕入先の市場名 |
| C | 品目 | 品名（年式・管理番号を除いたもの） |
| D | 特徴 | 色・サイズ・容量など（読み取れない場合は空白） |
| E | 型番 | 登録番号など（読み取れない場合は空白） |
| F | 仕入値 | 仕入れ価格（数値のみ、円マーク・カンマなし） |

年式は不要。数量の列も不要。

出力例：
```
年月日        場所           品目              特徴      型番               仕入値
2026/03/23   石川道具市場   バナニ槽式洗濯機   5kg                          8000
2026/03/23   石川道具市場   アクア1ドア冷蔵庫  シルバー  T4120001227774     3000
2026/03/23   石川道具市場   ヤマダサイコロ冷蔵庫                            2000
2026/03/23   石川道具市場   東芝クリーナー                                  2000
2026/03/23   石川道具市場   LPコンロ                    T7220001011513     5000
```

## 仕入れ伝票の実際の形式

### 伝票ヘッダー情報
- 日付（左上に記載）
- 仕入先（右上に記載、「株式会社」などは省略して市場名のみ）

### 品名リストの行フォーマット

品名の先頭には「管理番号」と「年式」が混在した形式で記載されます。

```
[管理番号(省略可)] [年式'] [品名] [特徴・色・サイズなど]
```

実例：
```
2 22' バナニ槽式洗濯機 5kg       → 品目: バナニ槽式洗濯機  特徴: 5kg
7 22' アクア1ドア冷蔵庫 シルバー  → 品目: アクア1ドア冷蔵庫  特徴: シルバー
21' LPコンロ                     → 品目: LPコンロ  特徴: (空白)
リンサークリーナー 布団乾燥機×2   → 品目: リンサークリーナー 布団乾燥機  特徴: ×2
シーリング×8                     → 品目: シーリング  特徴: ×8
24' タイガージャー 黒             → 品目: タイガージャー  特徴: 黒
```

### 年式の扱い（重要）
年式（22'、21'など）は出力に含めない。品目名と特徴の抽出にのみ使用し、列には出力しないこと。

### 管理番号の扱い
品名先頭の単独の数字（例: 2, 7, 34）は管理番号。品目名には含めないこと。

## Gemini API プロンプト

extractor.py 内で使用するプロンプト：

```
あなたはリサイクルショップの仕入れ伝票を読み取るAIアシスタントです。
この画像は仕入れ市場で発行された「買伝票」です。

以下のルールに従って、全商品をJSON配列形式で返してください。

【品名の読み取りルール】
- 「2 22' バナニ槽式洗濯機 5kg」の場合
  → 先頭の数字(2)は管理番号なので除外
  → 「22'」は年式なので除外
  → item_name = 「バナニ槽式洗濯機」
  → features = 「5kg」
- 色・サイズ・容量など特徴はfeaturesに入れること
- 「×2」「x2」「×8」などの数量表記はfeaturesに入れること
- 年式（22'など）は出力しないこと

【その他のルール】
- purchase_dateとlocationはヘッダーから読み取ること
- 仕入先は市場名のみ（「株式会社」などは省略）
- 型番（登録番号）はTから始まる英数字が多い。なければnull
- 読み取れない場合はnullを返すこと
- 合計金額行・税金行・参加費行は商品として含めないこと

返答は以下のJSON形式のみ（説明文不要）:
[
  {
    "purchase_date": "YYYY/MM/DD",
    "location": "市場名",
    "item_name": "品目名",
    "features": "特徴" または null,
    "model_number": "型番" または null,
    "purchase_price": 仕入れ値（整数）または null
  }
]
```

## Streamlit アプリのUI要件

### 画面構成

```
┌─────────────────────────────────────┐
│  🧾 仕入れ伝票 自動読み取りアプリ        │
├─────────────────────────────────────┤
│  [画像をアップロード]                   │
│  （JPG / PNG に対応）                 │
│  アップロード後に画像プレビューを表示     │
│                                      │
│  ［読み取り開始ボタン］                 │
├─────────────────────────────────────┤
│  読み取り結果                          │
│  仕入日: 2026/03/23                  │
│  仕入先: 石川道具市場                  │
│                                      │
│  | 品目 | 特徴 | 型番 | 仕入値 |       │
│  | ... | ... | ... | ...   |       │
│                                      │
│  合計仕入れ額: ¥106,000               │
├─────────────────────────────────────┤
│  ［Excelダウンロードボタン］            │
│  ファイル名: 20260323_石川道具市場.xlsx │
└─────────────────────────────────────┘
```

### 機能要件

1. 画像アップロード後、プレビューを表示する
2. 「読み取り開始」ボタンを押すとGemini APIに送信する
3. 処理中はスピナー（「読み取り中...」）を表示する
4. 結果はテーブルで表示する（年月日・場所は表の上に別表示でOK）
5. テーブルの下に合計仕入れ額を表示する
6. Excelダウンロードボタンで「YYYYMMDD_市場名.xlsx」形式で保存できる
7. エラー時はわかりやすいメッセージを日本語で表示する

## Excel出力の仕様

- ライブラリ: openpyxl
- ファイル形式: .xlsx
- 1行目はヘッダー行（年月日、場所、品目、特徴、型番、仕入値）
- 2行目以降がデータ
- 仕入値は数値型で出力（文字列にしないこと）
- 年月日はYYYY/MM/DD形式の文字列で出力

---

## 注意事項

- Geminiのモデルは「gemini-2.0-flash」を使用すること
- 画像は送信前に最大1600pxに縮小してJPEG圧縮すること
- エラーハンドリング: APIキー未設定・画像未選択・JSONパース失敗 の3つは必ず対処すること
- README.mdに日本語で起動手順を記載すること
